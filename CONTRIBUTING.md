# 贡献指南

感谢您对中小学编程教育平台知识体系项目的关注！我们欢迎任何形式的贡献。

## 如何贡献

### 报告问题

如果您发现了错误或有改进建议，请：

1. 在 [Issues](https://github.com/zhaotoday/k12-coding-education/issues) 页面搜索是否已有相关问题
2. 如果没有，创建一个新的 Issue
3. 清晰地描述问题或建议
4. 如果可能，提供截图或示例

### 提交贡献

1. **Fork 本仓库**
   ```bash
   # 点击页面右上角的 Fork 按钮
   ```

2. **克隆您的 Fork**
   ```bash
   git clone https://github.com/YOUR_USERNAME/k12-coding-education.git
   cd k12-coding-education
   ```

3. **创建特性分支**
   ```bash
   git checkout -b feature/your-feature-name
   # 或
   git checkout -b fix/your-bug-fix
   ```

4. **进行修改**
   - 保持文档格式一致
   - 遵循中文排版规范（中英文之间加空格）
   - 确保链接有效
   - 检查拼写和语法

5. **提交更改**
   ```bash
   git add .
   git commit -m "描述您的更改"
   ```

6. **推送到您的 Fork**
   ```bash
   git push origin feature/your-feature-name
   ```

7. **创建 Pull Request**
   - 访问原仓库页面
   - 点击 "New Pull Request"
   - 选择您的分支
   - 填写 PR 描述
   - 提交 PR

## 贡献内容类型

### 1. 补充资源
- 新的学习平台或工具
- 优质教程链接
- 开源项目推荐
- 书籍推荐

### 2. 更新信息
- 修正过时的信息
- 更新链接
- 更新版本号
- 更新数据统计

### 3. 修正错误
- 拼写错误
- 语法错误
- 格式问题
- 链接失效

### 4. 改进文档
- 优化结构
- 补充说明
- 添加示例
- 改进排版

### 5. 翻译
- 英文资源的中文说明
- 补充国际资源

## 文档规范

### 中文排版

1. **中英文之间加空格**
   - ✅ 正确：使用 React 开发前端应用
   - ❌ 错误：使用React开发前端应用

2. **中文与数字之间加空格**
   - ✅ 正确：适合 8-12 岁儿童
   - ❌ 错误：适合8-12岁儿童

3. **使用全角中文标点**
   - ✅ 正确：你好，世界！
   - ❌ 错误：你好,世界!

4. **专有名词使用正确的大小写**
   - ✅ 正确：GitHub、JavaScript、Python
   - ❌ 错误：github、javascript、python

### Markdown 格式

1. **标题层级**
   ```markdown
   # 一级标题
   ## 二级标题
   ### 三级标题
   ```

2. **链接格式**
   ```markdown
   [链接文字](https://example.com/)
   ```

3. **代码块**
   ````markdown
   ```python
   print("Hello, World!")
   ```
   ````

4. **列表**
   ```markdown
   - 无序列表项 1
   - 无序列表项 2
   
   1. 有序列表项 1
   2. 有序列表项 2
   ```

### 内容要求

1. **准确性**
   - 确保信息准确
   - 链接有效
   - 版本号正确

2. **完整性**
   - 提供必要的背景信息
   - 包含相关链接
   - 适当的分类

3. **客观性**
   - 避免主观评价
   - 提供事实依据
   - 平衡多方观点

4. **时效性**
   - 使用最新信息
   - 注明更新日期
   - 及时更新过时内容

## 提交信息规范

使用清晰的提交信息：

```bash
# 格式
<类型>: <简短描述>

# 示例
docs: 添加 Vue 3 学习资源
fix: 修正 React 官方文档链接
update: 更新竞品分析数据
refactor: 重构知识体系结构
```

### 提交类型
- `docs`: 文档更新
- `fix`: 修正错误
- `update`: 更新信息
- `add`: 添加新内容
- `remove`: 删除内容
- `refactor`: 重构文档结构
- `style`: 格式调整

## Review 流程

1. 提交 PR 后，维护者会进行 Review
2. 可能会提出修改建议
3. 根据反馈进行调整
4. 通过 Review 后会被合并
